1. Tổng quan

claude-code-anyllm là bridge giúp dùng Claude Code (VS Code extension) với bất kỳ LLM nào hỗ trợ OpenAI API — FPT Cloud, NVIDIA, Gemini, Groq, GitHub Models, OpenRouter, Ollama, v.v. Một local LiteLLM proxy dịch giữa Anthropic API (Claude Code) và OpenAI API (provider). 

VS Code (Claude Code)  ──►  LiteLLM proxy :4000  ──►  provider /v1 endpoint
  • Hỗ trợ Windows (PowerShell) và macOS/Linux (bash)
  • Không cần real Anthropic API key
  • Key không bao giờ lưu vào file — truyền qua env var
  • Response caching tự động (in-memory, 1h TTL)
  • Usage log mỗi session ghi vào logs/usage.csv

2. Luồng hoạt động

  1. User chạy .\start-claude.ps1 (hoặc ./start-claude.sh)
  2. Script đọc config/providers.conf, hiển thị menu chọn provider
  3. Resolve API key từ env var (LLM_API_KEY_FPT, LLM_API_KEY_GROQ, v.v.)
  4. Generate config/litellm_config.yaml (kèm cache settings)
  5. Start LiteLLM proxy trong background / cửa sổ mới
  6. Đợi proxy ready (max 30s), set env vars ANTHROPIC_*
  7. Mở VS Code ở project dir; ghi entry vào logs/usage.csv

5. Scripts chính

start-claude.ps1 / start-claude.sh

Main launcher. Các tính năng chính:

  • Provider resolution: flag -Provider, hoặc interactive menu từ providers.conf
  • Key resolution: inline → env var → prompt hidden input
  • Mode -CheckKeys: test GET /models với mỗi key, báo 200/401/429/404
  • Mode -Benchmark: 3 test prompts — Ping / Code gen / Tool-call check
  • Mode -Dashboard: mở browser tới http://localhost:PORT/ sau khi proxy ready
  • YAML gen bao gồm litellm_settings.cache: local, ttl=3600
  • Usage log: append logs/usage.csv sau mỗi session thành công
  • Better errors (bash): khi proxy timeout → đọc log, detect 401/429/ImportError

open-with-claude.ps1 / open-with-claude.sh mới

Template launcher copy vào project. Tự động set -OpenDir $PSScriptRoot và forward @args.

toggle-brain.ps1 / toggle-brain.sh

Chuyển Claude Code giữa real Anthropic và proxy mode bằng cách copy profile vào ~/.claude/settings.json.

setup-litellm.ps1 / setup-litellm.sh

Tạo Python venv, cài LiteLLM, pin Claude Code CLI sang stable channel.

6. Docker mới

Chạy proxy trong container — không cần cài Python trên máy host. Dùng cho team server, NAS, CI.

  • Dockerfile: base python:3.12-slim, cài litellm, expose port 4000
  • docker-entrypoint.sh: generate config từ env vars rồi start proxy
  • docker-compose.yml: pre-configured cho Groq, có comment blocks cho FPT/Gemini/NVIDIA
docker compose up -d
# hoặc:
docker run -d -p 4000:4000 -e LLM_API_KEY="..." -e BASE_URL="..." -e MODEL="..." claude-bridge

Env vars Docker: LLM_API_KEY, BASE_URL, MODEL, PORT (mặc định 4000).
