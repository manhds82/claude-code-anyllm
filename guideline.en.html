1. Overview

This toolkit lets you use Claude Code in VS Code as usual, while the actual "brain" answering you is any OpenAI-compatible LLM — FPT Cloud, OpenAI, OpenRouter, Groq, DeepSeek, a local Ollama, etc. The bridge is LiteLLM, a tiny proxy running on your own machine that translates between the Anthropic API and your provider.

➤
Data flow: VS Code (Claude Code) → LiteLLM localhost:4000 → your provider's /v1 endpoint
🖥

VS Code + Claude Code

The UI you type in. Pointed at the proxy via environment variables; no Anthropic key needed.

🔀

LiteLLM proxy

Runs locally on port 4000. Accepts Anthropic-style requests, forwards them to your provider, and translates responses back.

☁

Your provider

Where the real model runs. You only need a base URL, a model, and an API key.

The whole thing is just 2 scripts and 1 auto-generated config. Install once, run once.

2. Requirements

  • Windows (PowerShell), macOS, or Linux (bash).
  • VS Code with the Claude Code extension (or the Claude Code CLI).
  • Python 3.11 or 3.12 — check with py -3.12 --version. Get it at python.org (tick Add to PATH).
  • An API key and base URL for any OpenAI-compatible provider.

3. Install (once)

Open a terminal in the project folder and run the command for your OS:

# Windows (PowerShell)
.\setup-litellm.ps1

# macOS / Linux (bash)
chmod +x *.sh
./setup-litellm.sh

It finds Python, creates a .venv inside the project, installs litellm[proxy], and verifies it. Reinstall from scratch any time with .\setup-litellm.ps1 -Force (or ./setup-litellm.sh --force).

✓
Because .venv lives in the project, it does not depend on your username — copy the whole folder to another machine and it still works.
⚠
If PowerShell blocks the script (execution policy): run Set-ExecutionPolicy -Scope Process Bypass and try again.
📌
Stable Claude Code: setup also runs claude install stable once (when the claude CLI is on your PATH) to pin Claude Code to the stable channel — newer builds sometimes misbehave behind a proxy. The pin sticks across auto-updates; re-run it any time. Same command on every OS. Needs the standalone CLI (the VS Code extension alone won't have it).

4. Run

cd path/to/claude-code-anyllm

# Windows (PowerShell)
.\start-claude.ps1

# macOS / Linux (bash)
./start-claude.sh

The script starts the proxy (a separate window on Windows; the background on macOS/Linux, logging to litellm-proxy.log), waits until it is ready, then opens VS Code pointed at the proxy. Stop the proxy: close the proxy window, or run -Stop (Windows) / --stop (macOS/Linux). The parameters below use the Windows form (-Model); on macOS/Linux use the bash form (--model, --key, --stop, ...).

Parameters of start-claude.ps1
ParameterWhat it does
-BaseUrl "url"Use a different endpoint for this run.
-Model "name"Use a different model for this run (no file edit). E.g. -Model "Qwen3-32B"
-Key "key"Use a different key for this run (not saved to disk).
-ListList the models the endpoint exposes, then exit.
-Port 4010Change the proxy port (default 4000; auto-advances if busy).
-StopStop a running proxy on that port.
-NoVSCodeOnly start the proxy; print env commands so you can open VS Code yourself.
ℹ
On the first run, if LiteLLM isn't installed, the script auto-runs setup-litellm.ps1 for you — so you really only need to remember start-claude.ps1.

5. ★ Where to change the API KEY

Three ways — pick one:

📌

Option 1 — Permanent

Open start-claude.ps1 and set:
$Key = "your-key"

⚡

Option 2 — One run

No file edit, pass it at runtime:
.\start-claude.ps1 -Key "your-key"

🔒

Option 3 — Safest

Leave $Key = "". Each run the script asks for the key (hidden input, never written to disk).

⚠
Don't paste the key into config/litellm_config.yaml. That file is regenerated every run; the real key is loaded via the LLM_API_KEY environment variable the script sets.

6. ★ Where to change the MODEL

The single place to change the model is the $Model line at the top of start-claude.ps1:

# EDIT THESE FOR YOUR PROVIDER
$BaseUrl = "https://mkp-api.fptcloud.com/v1"   # your provider endpoint
$Model   = "DeepSeek-V4-Flash"                 # <-- change the model here
$Key     = ""                                  # blank = prompt at runtime

Or change it without editing the file:

.\start-claude.ps1 -List               # see which models exist
.\start-claude.ps1 -Model "Qwen3-32B"  # use that model for this run
★
Pick a model that supports tool / function calling — otherwise Claude Code can only reply with text, not read or edit files. Test it: ask it to "read README.md and summarize" or "create a file test.txt containing hello". If it touches the file, you're good; if it only talks, pick another model.
ℹ
To switch provider entirely, change $BaseUrl too (or pass -BaseUrl). You never need to touch $ClaudeAlias or the YAML — the script keeps them in sync.

7. ★ Brain Toggle — persistent profile switching

Unlike start-claude.ps1 which sets environment variables for one session, toggle-brain.ps1 writes a persistent profile to ~/.claude/settings.json. The change survives VS Code restarts, tab Code sessions, and system reboots.

➤
One script, both directions. No more managing two separate scripts. toggle-brain.ps1 handles switching to Claude, switching to proxy, status checks, proxy lifecycle, and key management — all in one place.
toggle-brain commands
I want toWindowsmacOS / Linux
Check current brain.\toggle-brain.ps1 -Status./toggle-brain.sh status
Switch to real Anthropic.\toggle-brain.ps1 -Mode claude./toggle-brain.sh claude
Switch to proxy LLM.\toggle-brain.ps1 -Mode proxy./toggle-brain.sh proxy
Switch + auto-start proxy.\toggle-brain.ps1 -Mode proxy -StartProxy./toggle-brain.sh proxy --start
Switch + start + set key.\toggle-brain.ps1 -Mode proxy -StartProxy -Key "sk-..."./toggle-brain.sh proxy --start --key "sk-..."
Stop the proxy.\toggle-brain.ps1 -StopProxy./toggle-brain.sh stop
List available profiles.\toggle-brain.ps1 -ListProfiles./toggle-brain.sh profiles

How profiles work

Each file in the profiles/ folder is a JSON snippet that sets Claude Code's environment variables. Switching = copying the chosen profile to ~/.claude/settings.json.

# profiles/claude.json — empty env = real Anthropic
{"env": {}}

# profiles/fpt.json — routes through LiteLLM proxy
{"env": {
  "ANTHROPIC_BASE_URL": "http://localhost:4000",
  "ANTHROPIC_AUTH_TOKEN": "litellm-proxy",
  "ANTHROPIC_MODEL": "claude-sonnet-4-6",
  "ANTHROPIC_DEFAULT_HAIKU_MODEL": "claude-sonnet-4-6",
  "ANTHROPIC_DEFAULT_SONNET_MODEL": "claude-sonnet-4-6",
  "ANTHROPIC_DEFAULT_OPUS_MODEL": "claude-sonnet-4-6"
}}

The profiles/ folder is extensible — add any .json file with the same structure to create a new provider preset. The script auto-discovers them.

⚠
Remember to restart Claude Code (close tab / New session) after switching. The profile is only read at startup.

9. How it works

Useful for debugging — you normally never touch any of this.

  • Config is generated: each run rewrites config/litellm_config.yaml from $ClaudeAlias, $Model, $BaseUrl (UTF-8, no BOM). So don't edit that file by hand.
  • Key via env var: the config points to api_key: os.environ/LLM_API_KEY; the script sets LLM_API_KEY for the proxy window. The key never lands in a file.
  • Pointing Claude Code at the proxy via env vars: ANTHROPIC_BASE_URL (the proxy), ANTHROPIC_AUTH_TOKEN (a dummy token; the real key lives in the proxy), ANTHROPIC_MODEL = the alias.
  • Background models routed too: the script also sets ANTHROPIC_DEFAULT_HAIKU_MODEL / SONNET / OPUS to the same alias, so Claude Code's background tasks don't call a model the proxy doesn't have.
  • Real Anthropic key removed from the session so Claude Code can't silently fall back to Anthropic.

10. Troubleshooting

Symptom → fix
SymptomFix
PowerShell blocks the scriptSet-ExecutionPolicy -Scope Process Bypass, then rerun.
Port 4000 is busyThe script auto-advances. Or choose one: -Port 4010.
Model only replies with text, won't edit filesThe model lacks tool calling — pick another (see section 6).
Proxy crashes with UnicodeDecodeErrorA stray .env with bad encoding — Rename-Item config\.env config\.env.bak.
Token-count errorsIn the run window: $env:CLAUDE_CODE_DISABLE_TOKEN_COUNTING = "1" before opening VS Code.
"Proxy ready" never appears after 30sRead the LiteLLM window for the real error (bad key, wrong model name, network).
"model not found" for haiku/sonnetMake sure you're on the current script (it routes background models to the same alias — section 8).
LiteLLM not foundRun .\setup-litellm.ps1 (or -Force to reinstall).
