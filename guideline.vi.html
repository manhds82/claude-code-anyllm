1. Tổng quan

Bộ công cụ này cho phép bạn dùng Claude Code trong VS Code như bình thường, nhưng phần "não" trả lời lại là bất kỳ LLM OpenAI-compatible nào — FPT Cloud, OpenAI, OpenRouter, Groq, DeepSeek, Ollama chạy local, v.v. Cầu nối là LiteLLM, một proxy nhỏ chạy ngay trên máy bạn, dịch qua lại giữa Anthropic API và provider của bạn.

➤
Luồng dữ liệu: VS Code (Claude Code) → LiteLLM localhost:4000 → endpoint /v1 của provider
🖥

VS Code + Claude Code

Giao diện bạn gõ lệnh. Được trỏ sang proxy bằng biến môi trường; không cần key Anthropic.

🔀

LiteLLM proxy

Chạy local ở cổng 4000. Nhận request kiểu Anthropic, chuyển tiếp sang provider rồi dịch ngược kết quả.

☁

Provider của bạn

Nơi model thật chạy. Bạn chỉ cần base URL, model và API key.

Toàn bộ chỉ gồm 2 script và 1 file config tự sinh. Cài 1 lần, chạy 1 lệnh.

2. Yêu cầu

  • Windows (PowerShell), macOS, hoặc Linux (bash).
  • VS Code đã cài extension Claude Code (hoặc Claude Code CLI).
  • Python 3.11 hoặc 3.12 — kiểm tra bằng py -3.12 --version. Chưa có thì tải tại python.org (nhớ tick Add to PATH).
  • Một API key và base URL của bất kỳ provider OpenAI-compatible nào.

3. Cài đặt (chỉ 1 lần)

Mở terminal tại thư mục project rồi chạy lệnh tương ứng với hệ điều hành:

# Windows (PowerShell)
.\setup-litellm.ps1

# macOS / Linux (bash)
chmod +x *.sh
./setup-litellm.sh

Script tự tìm Python, tạo .venv ngay trong project, cài litellm[proxy] rồi kiểm tra. Cài lại từ đầu khi cần: .\setup-litellm.ps1 -Force (hoặc ./setup-litellm.sh --force).

✓
Vì .venv nằm trong project nên không phụ thuộc tên user — chép cả thư mục sang máy khác vẫn chạy.
⚠
Nếu PowerShell chặn chạy script (execution policy): chạy Set-ExecutionPolicy -Scope Process Bypass rồi thử lại.

4. Chạy

cd path/to/claude-code-anyllm

# Windows (PowerShell)
.\start-claude.ps1

# macOS / Linux (bash)
./start-claude.sh

Script bật proxy (cửa sổ riêng trên Windows; chạy nền trên macOS/Linux, ghi log litellm-proxy.log), đợi sẵn sàng, rồi mở VS Code đã trỏ sang proxy. Tắt proxy: đóng cửa sổ proxy, hoặc chạy -Stop (Windows) / --stop (macOS/Linux). Các tham số bên dưới ghi theo dạng Windows (-Model); trên macOS/Linux dùng dạng bash (--model, --key, --stop, ...).

Tham số của start-claude.ps1
Tham sốTác dụng
-BaseUrl "url"Dùng endpoint khác cho lần chạy này.
-Model "name"Dùng model khác cho lần chạy này (không sửa file). VD: -Model "Qwen3-32B"
-Key "key"Dùng key khác cho lần chạy này (không lưu ra đĩa).

ℹ
Lần đầu chạy mà chưa cài LiteLLM, script tự gọi setup-litellm.ps1 giúp bạn — nên thực ra chỉ cần nhớ start-claude.ps1.

5. ★ Đổi API KEY ở đâu

Có 3 cách, chọn 1:

📌

Cách 1 — Vĩnh viễn

Mở start-claude.ps1 và đặt:
$Key = "key-cua-ban"

⚡

Cách 2 — Một lần

Không sửa file, truyền khi chạy:
.\start-claude.ps1 -Key "key-cua-ban"

🔒

Cách 3 — An toàn nhất

Để $Key = "". Mỗi lần chạy script hỏi key (gõ ẩn, không ghi ra đĩa).

⚠
Đừng dán key vào config/litellm_config.yaml. File đó tự sinh lại mỗi lần chạy; key thật được nạp qua biến môi trường LLM_API_KEY mà script tự đặt.

6. ★ Đổi MODEL ở đâu

Chỗ duy nhất để đổi model là dòng $Model ở đầu start-claude.ps1:

# EDIT THESE FOR YOUR PROVIDER
$BaseUrl = "https://mkp-api.fptcloud.com/v1"   # endpoint provider của bạn
$Model   = "DeepSeek-V4-Flash"                 # <-- đổi model ở đây
$Key     = ""                                  # để trống = hỏi lúc chạy

Hoặc đổi không cần sửa file:

.\start-claude.ps1 -List               # xem có model nào
.\start-claude.ps1 -Model "Qwen3-32B"  # dùng model đó cho lần này
★
Chọn model hỗ trợ tool / function calling — nếu không, Claude Code chỉ trả lời chữ chứ không đọc/sửa file được. Cách test: bảo nó "đọc README.md và tóm tắt" hoặc "tạo file test.txt nội dung hello". Làm được file = OK; chỉ nói suông = đổi model khác.
ℹ
Muốn đổi hẳn provider, sửa thêm $BaseUrl (hoặc truyền -BaseUrl). Không cần đụng $ClaudeAlias hay file YAML — script tự khớp.

8. Cơ chế hoạt động

Hữu ích khi debug — bình thường không cần đụng tới.

  • Config tự sinh: mỗi lần chạy ghi lại config/litellm_config.yaml từ $ClaudeAlias, $Model, $BaseUrl (UTF-8, không BOM). Vì vậy đừng sửa tay file này.
  • Key qua biến môi trường: config trỏ api_key: os.environ/LLM_API_KEY; script đặt LLM_API_KEY cho cửa sổ proxy. Key không nằm trong file nào.
  • Trỏ Claude Code sang proxy bằng env: ANTHROPIC_BASE_URL (proxy), ANTHROPIC_AUTH_TOKEN (token giả; key thật ở proxy), ANTHROPIC_MODEL = alias.
  • Route cả model nền: script đặt thêm ANTHROPIC_DEFAULT_HAIKU_MODEL / SONNET / OPUS về cùng alias, để tác vụ nền không gọi model mà proxy không có.
  • Xóa key Anthropic thật khỏi phiên để Claude Code không âm thầm fallback về Anthropic.
