1. Tổng quan

Bộ công cụ này cho phép bạn dùng Claude Code trong VS Code như bình thường, nhưng phần "não" trả lời lại là bất kỳ LLM OpenAI-compatible nào — FPT Cloud, OpenAI, OpenRouter, Groq, DeepSeek, Ollama chạy local, v.v. Cầu nối là LiteLLM, một proxy nhỏ chạy ngay trên máy bạn, dịch qua lại giữa Anthropic API và provider của bạn.

➤
Luồng dữ liệu: VS Code (Claude Code) → LiteLLM localhost:4000 → endpoint /v1 của provider
🖥

VS Code + Claude Code

Giao diện bạn gõ lệnh. Được trỏ sang proxy bằng biến môi trường; không cần key Anthropic.

🔀

LiteLLM proxy

Chạy local ở cổng 4000. Nhận request kiểu Anthropic, chuyển tiếp sang provider rồi dịch ngược kết quả.

☁

Provider của bạn

Nơi model thật chạy. Bạn chỉ cần base URL, model và API key.

Toàn bộ chỉ gồm 2 script và 1 file config tự sinh. Cài 1 lần, chạy 1 lệnh.

2. Yêu cầu

  • Windows (PowerShell), macOS, hoặc Linux (bash).
  • VS Code đã cài extension Claude Code (hoặc Claude Code CLI).
  • Python 3.11 hoặc 3.12 — kiểm tra bằng py -3.12 --version. Chưa có thì tải tại python.org (nhớ tick Add to PATH).
  • Một API key và base URL của bất kỳ provider OpenAI-compatible nào.

3. Cài đặt (chỉ 1 lần)

Mở terminal tại thư mục project rồi chạy lệnh tương ứng với hệ điều hành:

# Windows (PowerShell)
.\setup-litellm.ps1

# macOS / Linux (bash)
chmod +x *.sh
./setup-litellm.sh

Script tự tìm Python, tạo .venv ngay trong project, cài litellm[proxy] rồi kiểm tra. Cài lại từ đầu khi cần: .\setup-litellm.ps1 -Force (hoặc ./setup-litellm.sh --force).

✓
Vì .venv nằm trong project nên không phụ thuộc tên user — chép cả thư mục sang máy khác vẫn chạy.
⚠
Nếu PowerShell chặn chạy script (execution policy): chạy Set-ExecutionPolicy -Scope Process Bypass rồi thử lại.
📌
Claude Code bản ổn định: setup còn chạy claude install stable một lần (khi có claude CLI trong PATH) để ghim Claude Code vào kênh stable — bản mới đôi khi lỗi khi chạy qua proxy. Ghim này giữ nguyên qua các lần auto-update; chạy lại bất cứ lúc nào. Lệnh giống nhau trên mọi OS. Cần bản CLI standalone (extension VS Code không tự có).

4. Chạy

cd path/to/claude-code-anyllm

# Windows (PowerShell)
.\start-claude.ps1

# macOS / Linux (bash)
./start-claude.sh

Script bật proxy (cửa sổ riêng trên Windows; chạy nền trên macOS/Linux, ghi log litellm-proxy.log), đợi sẵn sàng, rồi mở VS Code đã trỏ sang proxy. Tắt proxy: đóng cửa sổ proxy, hoặc chạy -Stop (Windows) / --stop (macOS/Linux). Các tham số bên dưới ghi theo dạng Windows (-Model); trên macOS/Linux dùng dạng bash (--model, --key, --stop, ...).

4b. ★ Có nhiều key (FPT, NVIDIA, Gemini, GitHub...) — chọn provider khi chạy

Có key của nhiều nhà cung cấp cùng lúc? config/providers.conf liệt kê tất cả — chạy script không kèm tham số để hiện menu chọn, hoặc chỉ thẳng provider bằng -Provider/--provider. Mỗi provider đọc key từ biến môi trường riêng của nó, nên bạn chỉ cần đặt key cho cái nào bạn thực sự có.

# Windows (PowerShell)
.\start-claude.ps1                 # menu chọn, đánh dấu [key set]/[no key]
.\start-claude.ps1 -Provider nvidia
.\start-claude.ps1 -ListProviders   # xem hết id + biến môi trường tương ứng

# macOS / Linux (bash)
./start-claude.sh
./start-claude.sh --provider nvidia
./start-claude.sh --list-providers

✓
Thêm provider của riêng bạn: thêm 1 dòng vào config/providers.conf (dạng id|label|base_url|model|key_env) — không cần sửa code. -BaseUrl/-Model/-Key vẫn override provider đã chọn như trước, không gì thay đổi.

5. ★ Đổi API KEY ở đâu

Có 3 cách, chọn 1:

⭐

Khuyến nghị — biến môi trường

Để $Key = "" và đặt LLM_API_KEY trong môi trường; script tự đọc, key không nằm trong file nào.
setx LLM_API_KEY "key-cua-ban" (Windows)
export LLM_API_KEY="key-cua-ban" (macOS/Linux)

⚡

Một lần

Truyền khi chạy:
.\start-claude.ps1 -Key "..."
./start-claude.sh --key "..."

🔒

Hỏi mỗi lần

Để $Key = "" và không đặt env var — script hỏi key mỗi lần (gõ ẩn, không lưu).

⚠
Đừng bao giờ hard-code key vào start-claude.ps1/.sh hay config/litellm_config.yaml — các file này được git track, key thật ở đó có thể lộ lên GitHub. Hãy dùng biến môi trường ở trên.

6. ★ Đổi MODEL ở đâu

Chỗ duy nhất để đổi model là dòng $Model ở đầu start-claude.ps1:

# EDIT THESE FOR YOUR PROVIDER
$BaseUrl = "https://mkp-api.fptcloud.com/v1"   # endpoint provider của bạn
$Model   = "DeepSeek-V4-Flash"                 # <-- đổi model ở đây
$Key     = ""                                  # để trống = hỏi lúc chạy

Hoặc đổi không cần sửa file:

.\start-claude.ps1 -List               # xem có model nào
.\start-claude.ps1 -Model "Qwen3-32B"  # dùng model đó cho lần này
★
Chọn model hỗ trợ tool / function calling — nếu không, Claude Code chỉ trả lời chữ chứ không đọc/sửa file được. Cách test: bảo nó "đọc README.md và tóm tắt" hoặc "tạo file test.txt nội dung hello". Làm được file = OK; chỉ nói suông = đổi model khác.
ℹ
Muốn đổi hẳn provider, sửa thêm $BaseUrl (hoặc truyền -BaseUrl). Không cần đụng $ClaudeAlias hay file YAML — script tự khớp.

7. ★ Brain Toggle — chuyển não vĩnh viễn

Khác với start-claude.ps1 chỉ set biến môi trường tạm thời, toggle-brain.ps1 ghi profile vĩnh viễn vào ~/.claude/settings.json. Thay đổi tồn tại qua cả restart VS Code, đóng/mở tab Code, và reboot máy.

➤
Một script, cả hai chiều. Không cần quản lý hai file riêng. toggle-brain.ps1 xử lý chuyển sang Claude, chuyển sang proxy, kiểm tra trạng thái, quản lý proxy, và nhập key — tất cả trong một chỗ.
Lệnh toggle-brain
Tôi muốnWindowsmacOS / Linux
Xem não đang dùng.\toggle-brain.ps1 -Status./toggle-brain.sh status
Chuyển về Anthropic thật.\toggle-brain.ps1 -Mode claude./toggle-brain.sh claude
Chuyển sang proxy LLM.\toggle-brain.ps1 -Mode proxy./toggle-brain.sh proxy
Chuyển + tự bật proxy.\toggle-brain.ps1 -Mode proxy -StartProxy./toggle-brain.sh proxy --start

Xem danh sách profile.\toggle-brain.ps1 -ListProfiles./toggle-brain.sh profiles

Cơ chế profile

Mỗi file trong thư mục profiles/ là một JSON chứa biến môi trường cho Claude Code. Chuyển não = copy profile được chọn đè lên ~/.claude/settings.json.

# profiles/claude.json — env rỗng = Anthropic thật
{"env": {}}

# profiles/fpt.json — route qua LiteLLM proxy
{"env": {
  "ANTHROPIC_BASE_URL": "http://localhost:4000",
  "ANTHROPIC_AUTH_TOKEN": "litellm-proxy",
  "ANTHROPIC_MODEL": "claude-sonnet-4-6",
  "ANTHROPIC_DEFAULT_HAIKU_MODEL": "claude-sonnet-4-6",
  "ANTHROPIC_DEFAULT_SONNET_MODEL": "claude-sonnet-4-6",
  "ANTHROPIC_DEFAULT_OPUS_MODEL": "claude-sonnet-4-6"
}}

Thư mục profiles/ có thể mở rộng — thêm file .json với cấu trúc tương tự để tạo preset provider mới. Script tự động phát hiện.

⚠
Nhớ restart Claude Code (đóng tab / New session) sau khi chuyển. Profile chỉ được đọc khi khởi động.

9. Cơ chế hoạt động

Hữu ích khi debug — bình thường không cần đụng tới.

  • Config tự sinh: mỗi lần chạy ghi lại config/litellm_config.yaml từ $ClaudeAlias, $Model, $BaseUrl (UTF-8, không BOM). Vì vậy đừng sửa tay file này.
  • Key qua biến môi trường: config trỏ api_key: os.environ/LLM_API_KEY; script đặt LLM_API_KEY cho cửa sổ proxy. Key không nằm trong file nào.
  • Trỏ Claude Code sang proxy bằng env: ANTHROPIC_BASE_URL (proxy), ANTHROPIC_AUTH_TOKEN (token giả; key thật ở proxy), ANTHROPIC_MODEL = alias.
  • Route cả model nền: script đặt thêm ANTHROPIC_DEFAULT_HAIKU_MODEL / SONNET / OPUS về cùng alias, để tác vụ nền không gọi model mà proxy không có.
  • Xóa key Anthropic thật khỏi phiên để Claude Code không âm thầm fallback về Anthropic.
